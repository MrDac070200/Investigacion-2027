# Skill: Corporate Scientific Background Design

## Objective

Maintain a consistent visual identity throughout the application using backgrounds inspired by scientific and corporate presentations. These rules apply by default to all views, pages, and sliders, unless a specification explicitly indicates a different behavior.

---

# Design Principles

The background must never compete with the content.

The visual priority is:

1. Main content.
2. Interactive elements.
3. Decorative background.

Backgrounds must convey:

* Professionalism.
* Technology.
* Research.
* Modernity.
* Large and clean spaces.

---

# Default Background

## Base

* White background (#FFFFFF).
* A slightly bluish-white may be used when necessary.
* There must be abundant negative space.

## Decorative Shapes

Incorporate abstract organic shapes similar to waves or fluid blobs.

Characteristics:

* Soft edges.
* Natural curves.
* No predominant rigid shapes.
* Modern and elegant appearance.

Decorative shapes should be located primarily in:

* Top-left corner.
* Bottom-right corner.
* Sides of the view.

The center must always remain clear for content.

---

# Colors

Colors must come exclusively from the palette defined by the application.

Priority:

1. Dark blue.
2. Corporate blue.
3. Bright blue.
4. Light blue.

Transitions between colors must be achieved through smooth gradients.

Do not use:

* Intense reds.
* Predominant greens.
* Saturated colors outside the palette.

---

# Depth

The following may be added:

* Soft blur.
* Transparencies.
* Overlapping layers.
* Diffuse shadows.

The goal is to generate depth without distracting the user.

---

# Technological Elements

Optionally, the following may be added:

* Fine geometric lines.
* Semi-transparent circles.
* Soft particles.
* Abstract patterns.
* Light glassmorphism effects.

All these elements must have low opacity.

---

# Distribution

## Hero Sections

* Large shapes.
* Greater depth.
* Cinematic composition.
* Clear center.

## Sliders

* Softer backgrounds.
* Fewer elements.
* Prioritize readability.

## Forms

* Minimalist backgrounds.
* Decorations strictly in corners.

## Dashboards

* Off-white backgrounds.
* Very subtle abstract elements.

## Cards and Modals

* No complex decorative backgrounds.
* Use only neutral surfaces.

---

# Restrictions

Never:

* Use photographic images as backgrounds.
* Saturate the screen with decorative elements.
* Reduce content legibility.
* Place shapes behind important text.
* Create high-contrast compositions.

---

# Reference Style

Inspirations:

* Microsoft Fluent Design.
* Institutional scientific presentations.
* Modern dashboards.
* Soft glassmorphism.
* Corporate Tech.
* Academic Design.

---

# Default Rule

All views, pages, sliders, and sections must use this specification by default.

Only an explicit specification can overwrite these rules.

If a view does not define its own background, it must automatically inherit this style.
