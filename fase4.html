1. Conceptos fundamentales

🔌 ¿Qué es un bit?

Una computadora solo entiende dos estados: encendido (1) y apagado (0). Eso es un bit. Con n bits se pueden representar 2n valores distintos:

  • 8 bits → 256 valores
  • 32 bits → ~4,300 millones de valores

🔢 ¿Qué es FP32?

Floating Point de 32 bits es el formato estándar para entrenar redes neuronales. Divide los 32 bits en:

  • 1 bit de signo
  • 8 bits de exponente
  • 23 bits de mantisa

Ocupa 4 bytes por número y alcanza ~7 decimales de precisión.

🚀 ¿Qué es INT8?

Integer de 8 bits solo puede almacenar 256 valores enteros, típicamente de −128 a +127. No tiene decimales, pero ocupa solo 1 byte y permite operaciones matemáticas mucho más rápidas en hardware móvil.

2. Matemática de la cuantización

Para comprimir un peso real R (en FP32) a un entero Q (en INT8), se calculan dos parámetros por capa durante la calibración:

📏 Escala (S)

Qué "salto" representa cada nivel entero. Se calcula a partir del rango real de valores observados.

🎯 Punto cero (Z)

Qué valor entero corresponde al 0.0 real. Permite representar valores positivos y negativos de forma balanceada.

Fórmula de cuantización
Q = round ( R S + Z )
Fórmula de recuperación
R′ = S · ( Q − Z )

🧮 Ejemplo numérico concreto

1

En una capa convolucional los pesos reales varían entre −1.2 y +2.8.

2

Rango total: 2.8 − (−1.2) = 4.0. Casillas disponibles: 256.

3

Escala: S = 4.0 / 256 = 0.015625.

4

Cuantizamos el peso w = 0.001738291 con punto cero Z = 128:

Q = round ( 0.001738291 0.015625 + 128 ) = 128
5

Recuperamos: R′ = 0.015625 × (128 − 128) = 0.0. El error de redondeo es insignificante frente a millones de conexiones.

3. ¿Por qué no se destruye el modelo?

🧠 La información está en la configuración

Una red neuronal "sabe" que una lesión es melanoma porque miles de pesos están configurados de cierta manera. No importa si un peso es 0.037 o 0.031; importa que sea positivo y pequeño en lugar de negativo y grande.

🎲 El entrenamiento ya fue ruidoso

Durante el entrenamiento el modelo toleró dropout, augmentations y gradientes ruidosos. Aprendió a ignorar pequeñas perturbaciones. La cuantización es solo otra perturbación más.

✅ La decisión final es discreta

Al final el modelo no dice "la probabilidad es 0.8473291", sino ≥ 0.5 → melanoma. Si FP32 dice 0.8473 e INT8 dice 0.8451, ambos dan la misma decisión.

4. Proceso paso a paso: PTQ Estática

Nuestro método implementa Post-Training Static Quantization (PTQ): no se reentrena el modelo, solo se observa y comprime.

Paso 1

Exportar a ONNX FP32

El modelo PyTorch se convierte al formato ONNX, un estándar intercambiable que no requiere PyTorch para ejecutarse.

Paso 2

Calibración con datos reales

Se pasan 15 batches de 32 imágenes (480 imágenes) por el modelo FP32 para medir rangos de activaciones y calcular S y Z por capa.

Paso 3

Quantizar a INT8

Con los parámetros calculados, se reemplazan pesos FP32 por INT8 y las operaciones se aceleran con aritmética entera.

Paso 4

Formato QDQ

El grafo inserta nodos Quantize-Dequantize (Q → Conv → DQ) que ONNX Runtime ejecuta con instrucciones vectoriales en CPU ARM o Intel.

5. Analogías para entender la cuantización

🗺️

El mapa de la ciudad

FP32 es un mapa topográfico con curvas cada 1 mm: precisísimo, pero pesa 4 kg. INT8 es el mismo mapa con curvas cada 10 m: no sabes si el bache mide 2 o 3 cm, pero llegas al mismo destino con un mapa de 1 kg.

🍲

La receta de cocina

FP32 dice: "Agrega 250.000000 ml de agua". INT8 dice: "Un vaso de agua". La sopa sale igual de rica porque la receta no depende del miligramo exacto.

🎹

El piano

FP32 es un piano digital con 1,000 niveles de presión por tecla. INT8 tiene 256 niveles. Para el oído humano sigue siendo música; para una red neuronal que clasifica imágenes, no nota la diferencia.

👵

La báscula de la abuela

La abuela no necesita una báscula de laboratorio que diga 23.472891 kg. Con saber ~23 kg le basta para comprar la ropa. La red neuronal entrenada ya es experta; no necesita 6 decimales para decidir.

📉 Preservación de calidad

AUC valid: 0.8777 (FP32) → 0.8781 (INT8). Diferencia de solo +0.0004. La cuantización no degrada la capacidad diagnóstica del modelo.

⚡ Velocidad en hardware objetivo

En dispositivos móviles ARM con NPU/DSP, INT8 es el formato nativo. Un Snapdragon 8 Gen 3 ejecuta INT8 5-10× más rápido que FP32, con menor consumo de batería.

PTQ vs QAT

Enfoque Descripción Ventaja Desventaja
QAT Reentrena el modelo simulando INT8 durante el forward. Más preciso Requiere días de computación adicional
PTQ Toma el modelo entrenado, lo calibra en minutos y lo comprime. Rápido y simple Pérdida mínima de precisión (<0.5% AUC)
Justificación de ingeniería: Dado que el objetivo es desplegar en móviles con recursos limitados y el dataset de calibración es representativo, PTQ estático ofrece el punto óptimo entre simplicidad de implementación, tiempo de desarrollo y preservación de métricas clínicas.
